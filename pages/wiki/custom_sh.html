You will need to start by creating a file under assets/syntax/ called something like language-syntax.json. This JSON file will define what regex syntax corresponds to what colour.


JVM Languages need a Java Plugin to function correctly, but some basic syntax highlighting can still be done.


Inside the JSON file, we need to define a few things. First, we need to tell the program what file extension to associate this json with. To do that, we give the key "extension" the value of the extension, so in the case of java files, this would be "extension": "java"


Next, we need to define the path for the css. To do that, we add "css_path": "styles/syntax/mylanguage". For example, for java, this would be "css_path": "styles/syntax/java".


Finally, we can add the rules. To do that, create an array under the rules key: "rules": [], inside of which we can add our rules. To add a rule, we can open curly brackets, inside of which we need to define two values: regex and type.


In the regex key, we need to put the regular expression that matches the syntax rule. Make sure to escape any backslashes with a second backslash. (e.g. "\" becomes "\\". For the type key, put the css class that corresponds to that regex. (e.g. keyword). For more infos about regex, you can go here.


We can add as many of these rules as we want, the syntax json for JSON files can be seen below:

{
    "extension": "json",
    "css_path": "styles/syntax/json",
    "rules": [
        {
            "regex": "\"(\\\\.|[^\"])+\"(?=\\s*:)",
            "type": "key"
        },
        {
            "regex": "(null|true|false|-?\\d+\\.?\\d*|\"(\\\\.|[^\"])+\")(?=\\s*[,}\\]])",
            "type": "value"
        }
    ]
}
